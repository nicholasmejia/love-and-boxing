# Riddle Answer Carousel: Absolute Positioning with Single Transform Seam

The Answer Carousel (a sibling UI node `AnswerCarousel`, extracted from `RiddleBox` in May 2026) replaces the original `HBoxContainer` of three equal-sized cards with three free-floating Controls whose position, scale, and z-order are computed exclusively through `_compute_card_transform(index, rotation_state)` + `_make_position(anchor)` in `scripts/ui/answer_carousel.gd`. We chose this over the cheaper option of scaling cards in place inside an HBox because (a) the wrap-around rotation animation requires cards to leave the layout bounds and re-enter from the opposite side, which HBox does not permit, (b) a planned right-shift of the riddle box on screen needs the carousel to position itself independently of any layout container, and (c) a planned Phase C upgrade replaces the linear-wrap rotation with a tunable 3D orbit (perspective-projected angular positions with adjustable orbit-plane tilt), implemented as a function-body swap inside the same transform seam. All position math must route through those two functions — inlining anywhere else makes the Phase C swap structural rather than contained.
